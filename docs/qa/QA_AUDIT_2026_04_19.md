## Defects found & fixed

### ЁЯФ┤ Defect #1 тАФ Route-name collision тЖТ 500 on `/admin/archive`
- **Root cause:** `admin.campaigns.archive` was registered twice тАФ once for the *lifecycle* POST (`Active тЖТ Archived` status transition) and once for the new GET list of soft-deleted campaigns. Laravel's route-name map kept only the second one; when the hub page called `route('admin.campaigns.archive')` it bound to the POST and threw because the `{campaign}` param was missing.
- **Impact:** `/admin/archive` returns 500 the moment the hub is opened.
- **Fix:** renamed the soft-delete list route to `admin.campaigns.archiveList`; the trait now resolves the archive-list route via an override hook so campaigns don't collide with their own lifecycle name.
- **Regression guard:** full-page smoke covers `/admin/archive` тЖТ now 200.

### ЁЯЯа Defect #2 тАФ Permissions out of date in the live DB
- **Root cause:** the seeder was updated in a previous round to add `*.forceDelete` / `*.restore` for every module, but the installed DB was never re-seeded; `super_admin` still had only the 22 old permissions. This means a super_admin would hit 403 on `players.forceDelete`, `clubs.restore`, etc.
- **Fix:** ran `php artisan db:seed --class=RolesPermissionsSeeder`. super_admin now has 54 perms; none missing.
- **Production note:** deploy script must include `db:seed --class=RolesPermissionsSeeder --force` or an equivalent migration-driven role sync.

### ЁЯЯа Defect #3 тАФ Missing `public/storage` symlink
- **Root cause:** Laravel's public disk requires `php artisan storage:link` to be run once per install. It wasn't тАФ player photos, club logos, and the new platform logo would have returned 404 when served via `/storage/...`.
- **Fix:** ran `php artisan storage:link`. Now connected.
- **Production note:** add to deploy checklist (the `.env.production.example` already mentions it).

### ЁЯЯб Defect #4 тАФ Weak fallback initials
- **Root cause:** `Branding::initials()` took the first letter of each word, so `app_name="SFPA Voting"` rendered as "SV" тАФ not the branded "SFPA" the team expects when no logo is uploaded.
- **Fix:** the helper now detects that the first token is an all-caps acronym (2тАУ6 chars) and returns it verbatim (up to 4). Verified with 4 sample names:
  - `SFPA Voting` тЖТ **SFPA**
  - `FPA` тЖТ **FPA**
  - `Saudi Football Players Association` тЖТ `SFP`
  - `My Platform Name` тЖТ `MPN`
